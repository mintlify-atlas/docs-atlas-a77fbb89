Skills are named workflow programs stored as `SKILL.md` files that Codex loads on demand. Invoke any skill with `$skill-name "task description"` and OMX expands it into a full agent workflow. Browse all installed skills inside a Codex session with the `/skills` command.

<Info>
Skills are installed at `~/.codex/skills/*/SKILL.md` by `omx setup`. Each directory name is the skill name used in `$name` invocations.
</Info>

## Canonical workflow

The four core skills cover the full development cycle from vague idea to verified completion:

<Steps>
  <Step title="Clarify with $deep-interview">
    Run when the request or its boundaries are still unclear. The Socratic interview loop asks one question per round, scores ambiguity mathematically, and produces a spec artifact in `.omx/specs/` before any code is written.

    ```text
    $deep-interview "clarify the authentication change"
    ```

    Supports `--quick` (5 rounds, threshold 0.30), `--standard` (12 rounds, threshold 0.20, default), and `--deep` (20 rounds, threshold 0.15) depth profiles.
  </Step>

  <Step title="Plan with $ralplan">
    Runs Planner → Architect → Critic consensus planning. Produces canonical artifacts in `.omx/plans/` (PRD and test spec) that ralph and team require before execution.

    ```text
    $ralplan "approve the auth plan and review tradeoffs"
    $ralplan --deliberate "auth migration with pre-mortem"
    $ralplan --interactive "review each step with me"
    ```

    Add `--deliberate` for high-risk work (adds pre-mortem scenarios and expanded test planning). Add `--interactive` to approve each planning step manually.
  </Step>

  <Step title="Execute with $ralph or $team">
    Choose based on how the work should be distributed:

    - **`$ralph`** — persistent single-owner completion loop with architect verification at every iteration. Keeps working until task is fully verified or a hard blocker is hit.
    - **`$team`** — launches N coordinated Codex/Claude workers in tmux panes sharing `.omx/state/team/` state. Best for large approved plans that benefit from parallel lanes.

    ```text
    $ralph "carry the approved plan to completion"
    $team 3:executor "execute the approved plan in parallel"
    ```
  </Step>
</Steps>

### Utilities

| Skill | What it does |
|-------|-------------|
| `$cancel` | Cancel any active OMX mode (autopilot, ralph, ultrawork, team, …). The standard way to exit a mode after completion. |
| `$doctor` | Verify the OMX installation when something seems wrong. |
| `$help` | Show usage guidance from inside a session. |
| `$note` | Append a note to `.omx/notepad.md` for cross-session memory. |
| `$trace` | Inspect OMX runtime state and hook traces. |
| `$hud` | Surface HUD status from inside a session. |
| `$omx-setup` | Re-run setup to install or repair OMX components. |
| `$configure-notifications` | Interactive notification provider setup wizard. |
| `$ralph-init` | Initialize ralph state files without starting the loop. |
| `$skill` | Look up a skill's documentation. |
| `$worker` | Internal team-worker protocol and lifecycle (not for direct use outside `$team`). |

## How to invoke a skill

```text
$skill-name "task description or arguments"
```

Examples:

```text
$deep-interview "clarify the auth change"
$ralplan "approve the auth plan and review tradeoffs"
$ralph "carry the approved plan to completion"
$team 3:executor "execute the approved plan in parallel"
$code-review "review current branch"
$doctor
```

<Tip>
Some skills also activate via keyword detection without the `$` prefix. For example, typing "don't stop" triggers `$ralph`, and "fix build" triggers `$build-fix`. See the AGENTS.md keyword table for the full mapping.
</Tip>

## Browse skills

Inside any Codex session, type `/skills` to list all installed skills with their names and descriptions.
